02. Visual Hierarchy

How They Stack on a Screen

Level 1 Pill Tab — "Whose data?" (मेरे टिकट / टीम के टिकट)
↓ narrows to
Level 2 Underline Filter — "Which status?" (सभी / पेंडिंग / बंद / एक्सपायर्ड)
↓ further filters by
Level 3 Chips / Section Headers — "Which type?" (इंस्टॉलेशन + रिपेयर) or grouped in content
Rule: Max 2 filter levels on screen as UI controls. If a 3rd level is needed, use section headers in the content list — not another filter bar.
03. Decision Tree

Which Component to Use?

New filter/tab needed on a screen. Ask: 1. Does selecting this option change the ENTIRE dataset? (different source, different structure) → YES: Pill Tab → NO: Continue ↓ 2. Can only ONE option be active at a time? (statuses, states, single-category view) → YES: Underline Filter → NO: Continue ↓ 3. Can MULTIPLE options be active simultaneously? (combining attributes, additive filtering) → YES: Chips → NO: Rethink the interaction

06. Partner App Rules

Wiom-Specific Guidelines

  • 1
    List < 20 items → one filter level is enough. Don't over-engineer.
  • 2
    Underline filter count thresholds: 3-4 filters → fixed equal width. 5-6 filters → scrollable (horizontal scroll). 7+ filters → show 3-4 visible + More button → bottom sheet with remaining.
  • 3
    Only 2 options → use Pill Tab. Don't use Underline or Chips for binary choices — Pill handles it cleanly.
  • 4
    Pill is always optional → if screen shows only one dataset, skip it. Don't add a Pill just to have one.
  • 5
    Max 2 filter levels on screen → 3rd level = section headers in content, not more UI controls.
  • 6
    Underline overflow → More button opens bottom sheet with Radio select + Primary CTA. CTA disabled until selection.
  • 7
    Chips: selected = no border → filled state rule from stroke skill. Fill IS the boundary.
  • 8
    Pill above Underline → never reverse. Broader decision always above narrower.
  • 9
    Chips have minimal use cases currently → they exist to cover edge scenarios (e.g., NetBox Inventory) or when data grows enough to need multi-attribute filtering. Don't force Chips onto screens that don't need them today.
